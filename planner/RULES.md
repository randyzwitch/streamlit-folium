# Workspace Rules

## Page numbers
- `90-94`: shared workspace, samples, guides
- `95-99`: personal planner drafts and experiments

The existing `01-06` pages remain demo scenarios and should not be reused for planner experiments.

## File names
- Format: `95_screen_name.py`
- Example: `95_purchase_request_draft.py`
- One file should represent one screen draft.

## Do not add
- External API calls
- Database connections
- Secrets or real data
- Planner experiments inside the existing demo scenario files

## Preferred workflow
- Reuse the `planner` block functions and theme.
- Edit the constants and mock data first.
- Show on-screen summaries instead of real persistence.
- Keep copy short and task-oriented.

## Style rule
- Stay neutral and wireframe-like.
- Prefer cards, tables, forms, and buttons over decoration.
- Prioritize information structure and state visibility.
